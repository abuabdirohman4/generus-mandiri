## Key Decision Rules for Each Failing Test

When fixing a test assertion, check the actual function in `actions.ts`:

```
function returns {success: true, data: X}  → expect(result).toEqual({success: true, data: X})
function returns {success: false, message} → expect(result).toEqual({success: false, message: '...'})
function returns raw array                 → expect(result).toEqual(rawArray) OR expect(result).toHaveLength(N)
function throws Error                      → await expect(fn()).rejects.toThrow('message')
```

For `(result as any).data` on a plain array return: remove `.data`.
For `(result as any).data` on a `{success, data}` return: change to `result.data`.
